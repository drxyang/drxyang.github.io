** Introduction

*** Contents of Notebooks

- Microsoft Word documents
  + text
  + MATLAB commands
  + Output
- An interactive MATLAB session

*** Creating or Opening a MATLAB Notebook
    
- configuring Notebook (for Word 2010: =notebook -setup=)
- =notebook=
- converting a word document to notbook

*** Using Notebook

*Note*: 

- A good way to learn the various techniques in
  =matlabroot/notebook/pc/Readme.doc=.
- When you work on more than one MATLAB Notebook in a single word-processing session
  + Each notebook uses the same \"copy\" of MATLAB.
  + All notebooks share the same workspace.
  + You can protect the integrity of your workspace by the =clear=
    command as the first autoinit cell in the notebook.
- If you change a cell, it can contain inconsistent data: 
  + Input cells that depend on either the contents or the results of
    the changed cells do not automatically recalculate.
  + Consider selecting Evaluate MATLAB Notebook periodically
- Do not use debugging functions or the Editor while evaluating cells
  within a MATLAB Notebook
  1. Complete debugging files from within MATLAB.
  2. Clear all the breakpoints.
  3. Access the file using notebook.
  
**** Input cells and Output cells

1. Type the command into the MATLAB Notebook as text
2. Point in the command, then Notebook---Define Input Cell, or Alt-D
3. Alt-U to undefine cells
4. Notebook---Evaluate Cell, or Ctrl+Enter to get the output cell
5. Notebook---Purge Selected Output Cells, or press Alt+P to delete
   output cells.

**** Cell group

1. Use the mouse to select the input cells that are to make up the
   group
2. Notebook---Group Cells or press Alt-G.
3. Notebook---Evaluate Cell or press Ctrl+Enter to get output
4. Notebook > Evaluate MATLAB Notebook or press Alt+R.
5. Text or numeric output always comes first, regardless of the order
   of the commands in the group.

*Note*:

- A cell group cannot contain output cells.
- A cell group cannot contain text.
- If you select part or all of an output cell, but not its input cell,
  the cell group still includes the input cell.


**** Autoinit input cells

- It specify MATLAB commands be evaluated automatically each time a
  MATLAB Notebook opens. (quick and easy way to initialize the
  workspace)
- It is an input cells with the following additional characteristics:
  + The autoinit cells evaluate when MATLAB Notebook opens.
  + The commands in autoinit cells display in dark blue characters

**** Calc zones

- It partitions a MATLAB Notebook into self-contained sections
- It prepares problem sets, making each problem a separate calc zone
  that can be created and tested on its own. 
- A notebook can contain any number of calc zones.
- Notebook---Evaluate Cell or press Ctrl+Enter to get output

*** Printing and formatting a notebook

- Cell markers do not print.
- Input cells, autoinit cells, and output cells (including error
  messages) print according to their defined styles.

Reference: MATLAB help---MATLAB---User's Guide---Publishing MATLAB
Code

** Overview

*** Introduction

Publish formats:

  - HTML
  - XML
  - LaTeX
  - Microsoft Word

Contents can be published:

  - Commentary on the code, 
    + bulleted and numbered lists, 
    + bold and monospace font, 
    + preformatted text, 
    + LaTeX equations
  - MATLAB code
  - Results of running the code
    + output results
    + figures

*** Using code cells

- Any code cell features can be used for publishing
- Only comments at the start of a cell will be published 

*** Process for Publishing

1. Open the file in MATLAB Editor
2. Cell---Insert Text Markup
3. Publish the file
   - File---Publish /file name/: HTML published in =/html=
   - File---Publish Configuration for /file name/---Edit Publish
     Configurations for /file name/, then publish

*** Notes:

- To display the commented text as explanatory text, rather than
  MATLAB code, insert a cell break before the explanation.
- MATLAB publishes output generated by code immediately after the end
  of the cell that contains the code.
- create a section header without including a cell break: =%%%=

** Mark up MATLAB comments for publishing

*** Overview

- Aims of marking up
  + Polish the text file of code for publication
  + Single-source code with documentation
- Ways of marking up
  + Cell---Insert Text Markup (better for long commentary and
    descriptions)
  + Type the markup directly (better for short inline comments)

*** Document title and introductory text

At the beginning of the file

: %% DOCUMENT TITLE
: % INTRODUCTORY TEXT

Exported as =h1= in HTML (large size, bold font)

*** Title for the new section

At the beginning of the section

: %% SECTION TITLE
: % DESCRIPTIVE TEXT

Exported as =h2= in HTML

*** Preformatted Text

: %%
: %
: %  PREFORMATTED
: %  TEXT
: %

*NOTE*: DON'T delete the two spaces after =%=

Exported as text

- Monospaced
- Literal (maintains white space and does not wrap long lines)

*** Syntax highlighted sample code

: %%
: % 
: %   for i = 1:10
: %       disp x
: %   end
: % 

*NOTE*: DON'T delete the three spaces after =%=

*** Bulleted or numbered lists

: % * ITEM1
: % * ITEM2
: % * ITEM3 is a very long
: % one. NO problem

*NOTE*: To create a multiline list item, break the line as desired,
but do not insert the list item symbol (=*= or =#=) before the second
line.

*** External Graphics

: %% Image Example
: % This is a graphic:
: %
: % <<FILENAME.PNG>>
: %

*NOTE*: 

- FILENAME.PNG is relative to the publish folder
- Supported image types for output file formats

| Output file format | types of images you can use        |
|--------------------+------------------------------------|
| doc                | any format that MS Office supports |
| ppt                | any format that MS Office supports |
| html               | any format publishes successfully  |
| xml                | any format publishes successfully  |
| latex              | any format publishes successfully  |
| pdf                | bmp and jpg                        |

*** HTML markup

: % <html>
: % 
: % </html>

*NOTE*: only for HTML export

*** LaTeX markup

: % <latex>
: % 
: % </latex>

*NOTE*: only for LaTeX export

*** Inline LaTeX math equations

: % An inline equation: $x^2 + e^{\\pi i}$

*NOTE*: supports all output types for LaTeX math equations, except
Microsoft PowerPoint

*** LaTeX display math

: % $$x^2 + y^2 = 1$$

*** Force a snapshot of output

: %% Scale magic Data and Display as Image:
: 
: for i=1:3
:     imagesc(magic(i))
:     snapnow;
: end

*** Bold, italic, and monospaced text

: % *BOLD TEXT*
: % _ITALIC TEXT_
: % |MONOSPACED TEXT|

*** Trademark symbols

: % trademark symbol (TM)
: % registered tradmark symbol (R)

*** Hyperlinks

**** Overview

- HTML and XML export: dynamic or static hyperlinks
- MS Word: static hyperlinks

- dynamic hyperlinks
  + point the reader to MATLAB code or documentation
  + the link to run code
- static hyperlinks
  + link to a web location (URL)
  
You can include (print version) or exclude (web version) the URL
string from a static hyperlink.

**** Static hyperlinks and publishing URLs

: % <http://www.mathworks.com>

**** Static hyperlinks without publishing URLs

: % <http://www.mathworks.com MathWorks>

**** Dynamic hyperlinks

*NOTE*: need MATLAB running, not in Microsoft Word

***** Link to run code

: %% Recycling Preference
: % Click the preference you want:
: %
: % <matlab:recycle('off') Disable recycling>
: % <matlab:recycle('on') Enable recycling>

***** Link to a file

: %%
: % See the <matlab:web(fullfile(matlabroot,'toolbox','matlab','codetools','publish.m')) code> 
: % for the publish function.

***** Link to a MATLAB function reference page

: %%
: % See the help for the <matlab:doc('publish') publish> function.

***** Effect of copying hyperlinked text from the MATLAB window

Solution: cleaning up text markup before publishing

** Mark up MATLAB code for publishing

*** Overview

- Control the position of figure outputs
- interweave comments, code, and output to draw readers' attention to
  certain parts

*** Specifying the display of code output

- A cell break directs MATLAB to publish the code and output contained
  in that cells.
