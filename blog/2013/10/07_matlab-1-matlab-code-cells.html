MATLAB 内置的文学编程工具 1---MATLAB code cells

  •  开发环境搭建
  • MATLAB 配色和字体 

Reference: MATLAB User's Guide—Desktop Tools and Development Environment—Editing and Debugging MATLAB code—Evaluate Subsections of Files Using Code Cells

Introduction

Definition

A code cell contains the contiguous lines of code that are evaluated as a whole.

Usage

  • Evaluating sections of code in the Editor
    • experiment with code
    • debug
    • fine-tune
  • Publishing MATLAB files

Notes

  • MATLAB code files (.m files) only.
  • MATLAB considers the entire file to be a single cell; therefore, the first line in a file does not have to begin with %%.

Process for evaluating sections of files

  1. MATLAB Editor: Cell—Enable Cell Mode
  2. Define code budaries by %%
  3. Use cell features
    • Quick navigation
    • Evaluate the code in a cell
    • View results of a cell
    • Experiments with different values in cells

Define code cells

Implicit cell boundaries

  • Top and bottom of the file
  • Function declaration and function end statement (If you define an explicit cell break in a function)
  • Lines containing the start and end of the language construct (If you define an explicit cell break within a language construct)

Define explicit cells

  • Cell—Insert Cell Break
  • Click the Insert cell break button
  • %%
  • Select code lines and Cell—Insert Cell Breaks Around Selection

Cell titles

  • Text on the %% line called cell title. (following a space)

Highlight code cells

  • Current cell is highlighted by default

Turn off highlighting

  1. File—Preferences—Colors—Programming Tools
  2. Under Cell display options, clear Highlight cells

Set a highlighting color

  1. File—Preferences—Colors—Programming Tools
  2. Under Cell display options, select Highlight cells
  3. Select the color you like

Fix cell highlighting problems

Syntax errors brings problems to cell highlighting

This feature is turned on by

  1. File > Preferences > Code Analyzer
  2. Select Enable integrated warning and error messages.

Remove/Merge code cells

  1. Delete one % : the line changes to a standard comment
  2. Delete the entire %% line

Nested Cells

Associate Code Cell Breaks with Subfunctions

Place the cell break within the subfunction, rather than above the subfunction declaration. (Because of the implicit cell breaks by MATLAB)

Navigate among cells in a file

  • Normally use Ctrl-<Down> and Ctrl-<Up>
  • Go—Go to or Ctrl-<g> is used for large files

Evaluate cells

Evaluate code cells in a file

  • Run the cell code even if the file contains unsaved changes (good for debugging)
  • The file does not have to be on the search path.
  • The results display in the Command Window, figure window, or elsewhere.

Break points

  • breakpoints are ignored in cell evaluation. This means you cannot debug while running a single cell.
  • To use breakpoints, choose Run/Continue in the Debug menu.

Function files

  • You can define and evaluate cells in MATLAB function files (add the necessary variables to the base workspace)

Modify values in a cell

  • Using value modification tool in the cell toolbar to modify numbers in a cell can automatically re-evaluates the cell. (good for code experiment and fine-tunes)
  • You can also use (-, +, /, and *)
